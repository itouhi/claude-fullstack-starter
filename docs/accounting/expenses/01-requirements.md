## 1. 目的とスコープ

本モジュールは個人事業主が日常発生する経費支出を記帳し、未払計上・支払消込まで一元管理することを目的とする。電子帳簿保存法(電帳法)対応として、取引の証憑(領収書・請求書)をシステム内で保管・検索できる機能も提供する。

スコープ内:
- 経費の仕訳計上(即時支払い / 後払い)
- 証憑ファイルの添付・保管・検索
- 未払金の一覧管理と支払消込

スコープ外:
- 家事按分の計算 (M2 F-106 が担当。本モジュールは導線を提供するにとどまる)
- 取引先マスタの管理 (M1 F-203 が担当)
- 現金・預金の残高管理 (M5 担当)

## 2. 機能要件

### 2.1 F-401 経費入力

#### 2.1.1 基本的な機能

| REQ-ID | 要件 | 由来 |
|--------|------|------|
| REQ-401-01 | 経費科目(601〜690)を選択して経費仕訳を登録できる | F-401 |
| REQ-401-02 | 貸方科目として「現金(101)」「普通預金(102)」「未払金(305)」の3種を選択できる | F-401 |
| REQ-401-03 | 金額は円単位の整数で入力する | B-1 / 共通基盤 JournalLine |
| REQ-401-04 | 税区分を入力項目として保持する。既定値は選択した経費科目の `default_tax_code` を自動提案する | F-401 / N-5 |
| REQ-401-05 | インボイス登録番号(取引先の登録番号)を任意で入力・保持できる | N-5 / F-203 |
| REQ-401-06 | 取引先(Counterparty)を任意で紐づけられる | F-203 |
| REQ-401-07 | 摘要(取引の説明)を自由記述で入力できる | F-401 |
| REQ-401-08 | 取引日を入力する。既定値は本日日付 | F-401 |
| REQ-401-09 | 登録した経費仕訳は一覧表示・詳細参照できる | F-401 |
| REQ-401-10 | 登録した仕訳は訂正履歴を保持しながら修正できる(論理訂正) | F-103 / N-7 |

#### 2.1.2 入力項目一覧

| 項目名 | 型 | 必須 | 備考 |
|--------|-----|------|------|
| 取引日 | date | 必須 | 既定値: 本日 |
| 経費科目 | string (code) | 必須 | 601〜690から選択 |
| 金額 | integer (円) | 必須 | 正の整数 |
| 貸方科目 | string (code) | 必須 | 101/102/305から選択 |
| 税区分 | string (code) | 必須 | 経費科目の `default_tax_code` を自動提案 |
| 摘要 | string | 必須 | 最大 200 文字 |
| 取引先 | string (counterparty_id) | 任意 | M1 取引先マスタ参照 |
| インボイス登録番号 | string | 任意 | T + 13桁形式。電帳法要件 |

#### 2.1.3 税区分の自動提案ルール

| 経費科目 | 標準 default_tax_code | 自動提案税区分 |
|---------|----------------------|--------------|
| 租税公課(601) | EX | 対象外 |
| 水道光熱費(602) | TP10 | 課税仕入10% |
| 旅費交通費(603) | TP10 | 課税仕入10% |
| 通信費(604) | TP10 | 課税仕入10% |
| 接待交際費(605) | TP10 | 課税仕入10% |
| 損害保険料(606) | NT | 非課税 |
| 修繕費(607) | TP10 | 課税仕入10% |
| 消耗品費(608) | TP10 | 課税仕入10% |
| 減価償却費(609) | EX | 対象外 |
| 福利厚生費(610) | TP10 | 課税仕入10% |
| 外注工賃(611) | TP10 | 課税仕入10% |
| 地代家賃(612) | NT | 非課税 |
| 支払手数料(613) | TP10 | 課税仕入10% |
| 雑費(690) | TP10 | 課税仕入10% |

自動提案はあくまでデフォルト値であり、ユーザーが変更できる。

### 2.2 F-402 領収書添付

| REQ-ID | 要件 | 区分 | 由来 |
|--------|------|------|------|
| REQ-402-01 | 経費仕訳に対して証憑ファイル(画像 JPEG/PNG・PDF)を1件以上添付できる | 推奨 | F-402 / N-5 |
| REQ-402-02 | 添付する証憑に「取引日・金額・取引先」のメタ情報を記録する | 推奨 | N-5 / X3 |
| REQ-402-03 | 電帳法の検索要件として「取引日・金額・取引先」3キーでの証憑検索ができる | 推奨 | N-5 / X3 |
| REQ-402-04 | 証憑はサーバ上の指定ディレクトリに保存し、論理削除(DB 上の有効フラグ無効化)で管理する | 推奨 | N-5 / N-2 |
| REQ-402-05 | 証憑のアップロードは経費登録後に別途 API で行う(2ステップ登録) | 推奨 | F-402 |
| REQ-402-06 | 証憑ファイルの最大サイズは10MBとする | 推奨 | TBD (要確認) |

> **注記:** F-402 は「推奨」機能であるため、初期リリースでは証憑のメタ登録とファイル保存機能を実装し、検索機能は後続フェーズで詳細化する(REQ-402-03は TBD)。

### 2.3 F-403 未払金管理

| REQ-ID | 要件 | 区分 | 由来 |
|--------|------|------|------|
| REQ-403-01 | 貸方科目を「未払金(305)」として経費計上した仕訳を「未払計上」として識別できる | 任意 | F-403 |
| REQ-403-02 | 未払金残高のある仕訳(未消込)を一覧表示できる | 任意 | F-403 |
| REQ-403-03 | 未払金の支払消込として「未払金(305)借方 / 預金(101 or 102)貸方」の仕訳を登録できる | 任意 | F-403 |
| REQ-403-04 | 消込後の未払仕訳は「消込済み」として一覧から区別できる(補助科目または仕訳の source フィールドで管理) | 任意 | F-403 |
| REQ-403-05 | 未払金勘定(305)の残高は M7 試算表・BS で自動集計される(M4 独自管理不要) | 任意 | F-403 / B-1 |

> **注記:** 未払金は新エンティティを設けず、未払金勘定(305)の仕訳として表現する。未払残高の管理は仕訳ストアの残高計算(全体設計 §4.1)で導出する。

## 3. 非機能要件

| REQ-ID | 区分 | 要件 | 由来 |
|--------|------|------|------|
| REQ-N01 | 電帳法(N-5) | 証憑ファイルに取引日・金額・取引先を紐づけ、3キーによる検索を提供する | N-5 / X3 |
| REQ-N02 | 電帳法(N-5) | インボイス登録番号を仕訳明細レベルで保持する | N-5 |
| REQ-N03 | 操作ログ(N-7) | 経費仕訳の登録・訂正・削除(論理)はすべて監査ログ(X2)に記録する | N-7 |
| REQ-N04 | 操作ログ(N-7) | 証憑ファイルの添付・削除も監査ログに記録する | N-7 |
| REQ-N05 | 使いやすさ(N-9) | 税区分・貸方科目は前回入力値または科目マスタの既定値を自動補完し、入力ステップを最小化する | N-9 |
| REQ-N06 | 使いやすさ(N-9) | 経費入力フォームは簿記知識がない事業主でも操作できるよう、科目名称(日本語)で選択できる | N-9 |
| REQ-N07 | データ完全性 | 金額は正の整数のみ受け付け、仕訳のバランス(借方=貸方)は登録時にサーバ側で検証する | B-1 / 共通基盤 |
| REQ-N08 | データ完全性 | 論理削除・訂正は `EntryStatus` (ACTIVE/CORRECTED/DELETED) で管理し物理削除しない | N-2 / F-103 |
