## 設計の要点

- **売掛金の管理方法**: 専用エンティティを新設せず、勘定科目コード `135`(売掛金)の仕訳明細 (`JournalLine`) で完結。取引先の分離は `sub_account` (補助科目) で行う
- **未消込残高の定義**: `Σ借方(135/取引先) − Σ貸方(135/取引先)` = 仕訳ストアからの純粋な集計値
- **仕訳生成契約**: M2 `journal` の `POST /api/journal-entries` 相当を内部利用。M4〜M9 への依存なし
- **並列開発の位置づけ**: 全体設計 Layer 1。M2 基盤確立後に M4〜M6 と並行して開発可能
