# 要求事項定義書 — closing-tax (決算・確定申告・消費税)

## 1. 背景・目的

個人事業主は毎年2〜3月の確定申告期に、当該年度の取引をまとめた「青色申告決算書」を提出し、消費税課税事業者は消費税申告書を別途提出する必要がある。

本モジュール M8 `closing-tax` は以下を目的とする。

- **決算整理**: 期末時点での減価償却計上・家事按分・前払/未払の振替など、確定申告前に必要な整理仕訳を生成・管理する。
- **青色申告決算書の集約**: M6(固定資産)・M7(帳簿集計)の成果物を取り込み、青色申告決算書(損益計算書・貸借対照表・減価償却明細・月別売上)の各様式欄に対応するデータを提供する。
- **消費税納付額の算定**: 課税売上を税率別・事業区分別に集計し、簡易課税方式(サービス業=第五種、みなし仕入率50%)で消費税納付額を計算する。
- **申告用データ出力**: 確定申告に必要な数値をCSV/PDF形式でエクスポートし、税理士との共有や電子申告の参照資料として利用できるようにする。

## 2. スコープ

- **対象**:
  - 決算整理仕訳の種類・生成・管理 (F-801)
  - 青色申告決算書の出力データ集約 (F-802): 損益計算書・貸借対照表・減価償却明細・月別売上
  - 申告データのCSV/PDF出力 (F-803)
  - 消費税納付額計算(簡易課税・第五種) (F-804)
  - 課税事業者・インボイス登録事業者前提 (P-3)
  - 簡易課税制度適用事業者前提 (P-4)
  - 暦年(1/1〜12/31)固定の会計年度 (P-7)
  - 青色申告特別控除65万円適用(e-Tax届出・優良電子帳簿要件) (P-1)

- **対象外**:
  - 一般課税(原則課税)による仕入税額控除の計算
  - e-Tax への直接データ送信連携(当面CSV/PDF出力で代替。§9残課題)
  - 源泉徴収・所得税額の自動計算(税額計算は参考値のみ)
  - 複数事業区分の混在課税売上の自動按分(税区分マスタの `business_type` で拡張余地あり)
  - 法人税・法人の申告書類

## 3. ステークホルダー

| 役割 | 関心事 |
|------|--------|
| 個人事業主(主利用者) | 確定申告書類に必要な数字をシステムから引き出せること。手計算・転記ミスをなくしたい |
| 税理士(任意の関与先) | 確認・署名のために正確な数値と集計根拠を共有できること(CSV/PDF出力) |
| 税務当局(間接的) | 法令(青色申告・インボイス・電帳法)に準拠した記録・申告が行われていること |

## 5. 制約条件

- **技術スタック**: FastAPI (Python 3.14) + Vue 3 + Vite + TypeScript。既存スターターの規約を踏襲する (CLAUDE.md)。
- **仕訳モデル**: `backend/app/domain/journal.py` の `JournalEntry` / `JournalLine` を使用。金額は整数円(補助単位なし)。
- **税区分マスタ**: `backend/app/store.py` の `STANDARD_TAX_CATEGORIES` に定義済みの T10(課税売上10%)・T08(課税売上8%軽減)を使用する。
- **勘定科目**: 科目コード 330(未払消費税)・601(租税公課)・609(減価償却費)・160(前払費用)・305(未払金)・410(事業主貸)は `backend/app/store.py` に定義済み。
- **会計年度**: 暦年固定(P-7)。`fiscal_year` は整数(西暦)。
- **Layer依存**: M8 は Layer 3 に位置し、M6(固定資産)・M7(帳簿集計)の完成を前提とする(全体BD §6)。
- **e-Tax 直接連携**: 当面スコープ外。CSV/PDF 出力で代替する(全体BD §9 残課題)。

## 6. 前提・リスク

- **前提**:
  - 利用者は課税事業者かつインボイス登録事業者である (P-3)。
  - 利用者は簡易課税制度の届出を済ませており、事業区分は第五種(サービス業)のみである (P-4)。
  - M6(固定資産)・M7(帳簿集計)モジュールの主要APIが実装済みであること。
  - 仕訳入力時に税区分(T10/T08)と税額が正確に記録されていること。これが消費税集計の精度を左右する。
  - 青色申告65万控除はシステム外の手続き(e-Tax届出・優良電子帳簿届出)が完了していることを前提に金額を表示する。

- **リスク**:
  - **複数事業区分**: 将来的にサービス業以外の事業区分(第一〜第四種・第六種)を追加する場合、集計ロジックを分岐する必要がある。設計段階で拡張余地を確保する(REQ-032)。
  - **税率改正**: 消費税率・みなし仕入率の改正時は TaxCategory マスタ更新と集計ロジックの整合確認が必要(REQ-102)。
  - **M6/M7 未完**: 依存モジュールが未完成の場合、F-802の結合テストが実施できない。依存インターフェースをスタブで代替できる設計とする。
  - **端数処理**: 税率別に端数処理すると集計方法によって納付税額が1円ずれる場合がある。処理単位と切捨て方針をREQ-034で確定させる。
