# 要求事項定義書 — fixed-assets (固定資産・減価償却)

## 1. 背景・目的

個人事業主がパソコン・机・機器類などの業務用固定資産を取得した際、取得価額を即時に経費計上するのではなく、耐用年数にわたって分割計上する「減価償却」の仕組みが必要である。現状、手計算やエクセル管理が多く、計算誤りや台帳との不整合、決算時の計上漏れが頻発している。本モジュール (M6 `fixed-assets`) は固定資産台帳の登録・管理と年間償却費の自動計算・仕訳計上を担い、青色申告の少額減価償却資産特例 (30万円未満の即時償却) にも対応する。

由来: 要件定義書 F-601 (固定資産台帳) / F-602 (減価償却計算) / F-603 (少額特例) / P-1 (青色申告)

## 2. スコープ

- **対象**:
  - 業務用固定資産 (工具器具備品 科目コード150) の台帳登録・管理
  - 定額法による年間減価償却費の自動計算
  - 期中取得時の月割計算 (取得月から期末月まで)
  - 決算整理時の減価償却仕訳の自動生成 (M8 `closing-tax` との連携)
  - 家事按分(事業専用割合)による事業分の算出
  - 青色申告特例: 取得価額30万円未満の少額減価償却資産の即時償却 (F-603)
- **対象外**:
  - 定率法による減価償却 (当初バージョンでは対象外。TBD)
  - 土地・借地権 (非償却資産)
  - リース資産 (オペレーティングリース契約は経費直接計上)
  - 建物・構築物 (個人事業主・サービス業前提。要件定義第10章)
  - 固定資産の売却・除却・廃棄 (TBD: 今後の検討事項)

## 3. ステークホルダー

| 役割 | 関心事 |
|------|--------|
| 個人事業主 (主ユーザー) | 固定資産を登録するだけで年間の償却費が自動計算され、決算時に仕訳が自動生成されること。少額特例で即時経費化できる判定を自動でしてほしい |
| 税理士・記帳担当 | 定額法の計算式が法令どおり(取得価額×償却率)であること。期中取得の月割計算が正確であること。台帳と仕訳が一致していること |

### 機能要求

#### F-601 固定資産台帳

| ID | 要求内容 | 優先度 | 受け入れ基準 | 由来 |
|----|---------|--------|-------------|------|
| REQ-001 | 資産名・取得日・取得価額・耐用年数・償却方法・事業専用割合を入力して固定資産台帳に登録できること | Must | 入力後、台帳レコードが作成され、未償却残高の初期値が取得価額と等しくなる | F-601 |
| REQ-002 | 登録した固定資産の一覧を表示できること (資産名・取得日・取得価額・未償却残高・償却累計額) | Must | 一覧画面で全登録資産が表示される。未償却残高は最新値を反映する | F-601 |
| REQ-003 | 登録済みの固定資産の情報 (摘要・事業専用割合) を修正できること | Should | 修正後に台帳が更新される。取得価額・取得日・耐用年数の変更は監査ログに記録する | F-601 / N-7 |
| REQ-004 | 固定資産の償却予定表 (年度別の期首未償却残高・当期償却額・期末未償却残高) を表示できること | Must | 取得年度から完全償却年度 (備忘価額1円まで) の各年度が一覧表示される | F-601 |

#### F-602 減価償却計算

| ID | 要求内容 | 優先度 | 受け入れ基準 | 由来 |
|----|---------|--------|-------------|------|
| REQ-011 | 定額法により年間減価償却費を自動計算できること。計算式: 取得価額 × 償却率 (= 1 / 耐用年数、円未満切り捨て) | Must | システムが算出した年間償却費が、取得価額×(1/耐用年数)の切り捨てと一致すること | F-602 |
| REQ-012 | 期中取得の場合、取得月から事業年度末月までの月数で按分した月割り償却費を計算できること | Must | 取得月から12月末までの月数 / 12 で月割計算し、切り捨てた金額が年間償却費として算出される | F-602 |
| REQ-013 | 最終年度 (未償却残高が償却費以下になる年度) は備忘価額1円を残して償却すること | Must | 最終年度の償却額 = 期首未償却残高 − 1円 となり、未償却残高が1円で止まること | F-602 |
| REQ-014 | 事業専用割合 (0.01〜1.00) を指定した場合、年間償却費に事業専用割合を乗じた金額を事業経費とすること | Must | 事業専用割合0.8なら、算出年間償却費×0.8が減価償却費609として計上され、残0.2分は事業主貸410へ振替されること | F-602 / F-105 |
| REQ-015 | 減価償却の仕訳を計上できること。仕訳: 借方=減価償却費(609) / 貸方=工具器具備品(150) ※直接法 | Must | 仕訳計上後に工具器具備品(150)残高が減少し、減価償却費(609)が増加すること | F-602 |
| REQ-016 | 当期分の減価償却仕訳を計上すると、固定資産台帳の未償却残高と償却累計額が自動更新されること | Must | 計上後の未償却残高 = 計上前未償却残高 − 当期償却費 となること | F-602 |

#### F-603 少額減価償却資産特例

| ID | 要求内容 | 優先度 | 受け入れ基準 | 由来 |
|----|---------|--------|-------------|------|
| REQ-021 | 取得価額が30万円未満の場合、青色申告特例として即時全額償却 (取得年度に全額計上) を選択できること | Should | 少額特例を選択すると、取得年度の償却額が取得価額全額となり、翌年度以降の残高が0円 (備忘1円) となること | F-603 / P-1 |
| REQ-022 | 取得価額が30万円以上の場合、少額特例は選択できないこと (判定は自動で行い画面に明示すること) | Should | 取得価額が30万円以上で少額特例を試みた場合、エラーメッセージを表示して登録を拒否すること | F-603 |
| REQ-023 | 少額特例で即時償却した場合も、固定資産台帳に登録・管理し証憑と紐づけられること | Should | 即時償却後も台帳レコードが残り、取得価額・償却済み金額・備忘残高が確認できること | F-603 / N-5 |

## 5. 制約条件

- 金額は円単位の整数で管理する。小数・端数計算は円未満切り捨て (全体基本設計 §4 / B-2)
- 勘定科目コードは標準科目マスタの固定コードを使用する:
  - 工具器具備品 = 150 (資産 / AccountType.ASSET)
  - 減価償却費 = 609 (費用 / AccountType.EXPENSE)
  - 事業主貸 = 410 (純資産 / AccountType.EQUITY)
- 償却方法の初期対応は定額法のみ。定率法は将来対応 (TBD)
- 会計年度は暦年 (1月1日〜12月31日) 固定 (P-7 / 全体基本設計 §7)
- 仕訳の生成・保存は共通の仕訳エンジン (M2 `journal`) の契約 (`JournalEntry` / `JournalLine`) を利用する
- バックエンド: FastAPI (Python)。フロントエンド: Vue 3 + TypeScript (Composition API + `<script setup>`)
- 少額特例の適用条件: 青色申告事業者、取得価額30万円未満 (租税特別措置法第28条の2)

## 6. 前提・リスク

- **前提**:
  - M1 `core-masters` が勘定科目マスタ (150/609/410) と税区分マスタを提供済みであること
  - M2 `journal` の仕訳エンジン (`JournalEntry`/`JournalLine` のバランス検証) が稼働済みであること
  - 本システムのユーザーは青色申告事業者であること (P-1) — 少額特例の前提
  - M8 `closing-tax` (決算整理) が減価償却仕訳の計上エンドポイントを呼び出す連携を担う

- **リスク**:
  - 耐用年数省令の改定 (N-6) に対応するため、償却率をマスタ管理する設計が必要だが、当初はハードコードで対応し後続バージョンでマスタ化する可能性がある (TBD)
  - 家事按分割合が年度途中で変更された場合の取り扱いが未定 (TBD)
  - 固定資産の売却・除却 (残存簿価の損失計上) は本バージョン対象外だが、台帳設計に影響する可能性がある
